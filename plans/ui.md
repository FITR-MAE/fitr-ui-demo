# UI Reference

## Purpose

This document is a high-level reference for the layout, structure, and intended role of each major surface in the app.

It is meant for meetings and alignment, not implementation detail.

## App Structure

The app is organized around a small set of core product surfaces:
- Flow / Home
- Notifications
- Drops
- Studio
- Profile
- Search

Together, these surfaces should express the full product system: discovery, creation, intelligence, identity, and social interaction.

## Navigation

The bottom navigation should reflect the core user loop:
- consume
- respond
- create
- get guidance
- manage identity

Working navigation model:
- Home
- Notifications
- Drops
- Studio
- Profile

## Flow / Home

The home page is the feed and central discovery surface.

Its UI role is to feel fast, immersive, and signature to the brand. It should be the place for recommended fits, social discovery, aspiration, and daily inspiration.

Naming ideas explored:
- Flow
- Pulse

## Notifications

Notifications should act as a combined updates and communication page.

Its structure is split into:
- Activity
- Messages

Activity is for interactions, events, and alerts. Messages is for direct conversation.

## Drops

Drops is the creation surface.

At a high level, it should support two roles:
- creating new content
- managing wardrobe information

Over time, this page may become the operational home for clothing items and their metadata.

## Studio

Studio is the most strategic page in the app.

It is the product’s intelligence layer and should feel like the engine of the system. Its role is to connect personal context, AI guidance, daily recommendations, identity modeling, and social inspiration through similarity.

Working Studio structure:
- SŌEN
- Fits
- Style DNA
- Orbit

### SŌEN

The AI conversation layer.

This should feel like a personal fashion intelligence with memory, context, and a distinct point of view.

### Fits

The daily recommendation layer.

This is where the user sees how their style gets applied in practice.

### Style DNA

The identity and logic layer.

This is where the user’s style is described, interpreted, and broken down.

### Orbit

The inspiration and recommendation-through-similarity layer.

This is where users see adjacent people, overlapping style signals, and possible directions for evolution.

## Profile

Profile is the public-facing identity page.

Its role is to show the user as both a person and a style system. It should balance public identity, posts, wardrobe visibility, and private saved context where relevant.

## Search

Search is the utility discovery surface.

It should help users move across all searchable entities in the app: posts, users, clothes, brands, and stores. This surface is less about branding and more about clarity, structure, and filtering.

## Experience Model

At a high level, the app should feel like this:
- Flow shows what is happening now
- Drops lets users contribute and organize
- Studio explains and recommends
- Profile presents identity
- Search supports discovery
- Notifications keeps the social layer active

## Future Surfaces

Potential future UI surfaces include:
- year-end style report
- style from album or photo
- community style challenges

These should be understood as extensions of the same system, not separate products.
